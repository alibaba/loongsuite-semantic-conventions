## Overview

This document defines Alibaba-specific extensions to the [GenAI Rerank semantic conventions](../gen-ai-spans.md).
GenAI rerank represents a request to a Generative AI service that reorders documents based on their relevance
to a query. This is commonly used in RAG (Retrieval-Augmented Generation) pipelines to improve retrieval quality.
These extensions add Alibaba infrastructure context to rerank telemetry.

### Span name

**Span name** SHOULD be `rerank {gen_ai.request.model}` if model is available, otherwise `rerank`.

### Attributes

| Attribute | Type | Description | Requirement Level |
| --- | --- | --- | --- |
| `gen_ai.rerank.query` | string | The query used for reranking | Opt-in |
| `gen_ai.rerank.top_k` | int | Number of top documents to return | Recommended |
| `gen_ai.rerank.input_documents` | string[] | Input documents for reranking | Opt-in |
| `gen_ai.rerank.output_documents` | string[] | Output documents after reranking | Opt-in |

> [!Warning]
> `gen_ai.rerank.query` and document attributes may contain sensitive information.

### Alibaba Rerank Span Attributes

In addition to the standard [GenAI Rerank attributes](../gen-ai-spans.md), the following
Alibaba-specific attributes SHOULD be populated for rerank spans running on Alibaba infrastructure:

| Attribute | Type | Description | Requirement Level |
| --- | --- | --- | --- |
| `alibaba.base.env` | string | Environment identifier | Recommended |
| `alibaba.base.idc` | string | Data center identifier | Recommended |
| `alibaba.user.data` | string | User-defined custom data attached to the rerank call | Opt-in |
